Cloud Storage Setup

Connect DAVE to your cloud files in minutes. Pick a service below and follow the steps — no coding required.

Amazon S3 Best for large asset libraries already on AWS Google Drive Easiest option — just link your Google account

Amazon S3 Setup

About 10 minutes
1 Allow DAVE to talk to your S3 bucket (CORS)

By default, browsers block websites from reading files on other servers. You need to tell AWS "it's OK for DAVE to access this bucket." This is a one-time setting called CORS.

  1. Open the AWS S3 Console
  2. Click on your bucket name
  3. Go to the Permissions tab
  4. Scroll down to Cross-origin resource sharing (CORS) and click Edit
  5. Delete anything already there, then paste this:
[
  {
    "AllowedHeaders": [
      "Authorization",
      "x-amz-date",
      "x-amz-content-sha256",
      "content-type"
    ],
    "AllowedMethods": ["GET", "HEAD"],
    "AllowedOrigins": ["*"],
    "ExposeHeaders": [],
    "MaxAgeSeconds": 3600
  }
]
  1. Click Save changes

The "*" in AllowedOrigins means "allow any website." This is fine for getting started. Later you can lock it down to just https://drorlazar-sett.github.io and http://localhost:7777.

2 Get your AWS access keys

You need an Access Key ID and Secret Access Key — think of them like a username and password for programs to access your bucket.

Safety tip: Don't use your main AWS account keys. Create a separate user with read-only access to your bucket. That way even if the keys leak, nobody can delete or change your files.

If you already have keys, skip to Step 3. Otherwise:

  1. Open the AWS IAM Console
  2. Go to Users → Create user
  3. Give it a name like dave-viewer
  4. On the permissions step, click Attach policies directly and search for AmazonS3ReadOnlyAccess. Check the box and continue.
  5. After creating the user, go to Security credentials tab → Create access key
  6. Choose "Application running outside AWS"
  7. Copy both the Access Key ID (starts with AKIA...) and the Secret Access Key

Write down the Secret Access Key now — AWS won't show it again after you leave this page.

3 Enter your credentials in DAVE
  1. Open DAVE and click the Settings button in the top bar
  2. In the AWS S3 section, fill in:
    • Access Key ID — the one starting with AKIA...
    • Secret Access Key — your secret key
    • Region — the AWS region your bucket is in (e.g. eu-central-1)
    • Default Bucket — the name of your bucket
  3. Click Save

Not sure what region your bucket is in? In the AWS S3 console, click your bucket → Properties tab. The region is shown near the top (e.g. "EU (Frankfurt) eu-central-1").

4 Browse your files!

Click Source in the toolbar and choose AWS S3. A folder browser will open where you can navigate your bucket and load files into the DAVE grid.

Other ways to open S3 content:

  • Paste a URL — copy any S3 console URL from your browser and paste it into DAVE's search bar
  • Drag & drop — drag an S3 URL from your browser into the DAVE window
  • s3:// links — type s3://bucket-name/folder/ into the search bar

Google Drive Setup

About 5–10 minutes
1 Create a Google Cloud project

Google needs a "project" to manage your app's access. Don't worry — it's free and takes a minute.

  1. Go to Google Cloud Console
  2. If this is your first time, agree to the terms of service
  3. Click the project dropdown at the top → New Project
  4. Name it something like DAVE Viewer and click Create
  5. Make sure your new project is selected in the dropdown at the top
2 Turn on the Google Drive API
  1. In Google Cloud Console, go to APIs & Services → Library (in the left sidebar)
  2. Search for Google Drive API
  3. Click on it and press the big blue Enable button
3 Set up the consent screen

This tells Google what to show users when they sign in. You only need to do this once.

  1. Go to APIs & Services → OAuth consent screen
  2. Choose External and click Create
  3. Fill in the required fields:
    • App name: DAVE Viewer (or anything you like)
    • User support email: your email
    • Developer contact email: your email
  4. Click Save and Continue through the remaining steps (Scopes, Test users — you can skip these for now)

Since the app is in "Testing" mode, only your Google account (and accounts you add as test users) can sign in. This is perfectly fine for personal use.

4 Create your OAuth Client ID

This is the key piece DAVE needs — a Client ID that lets the sign-in popup work.

  1. Go to APIs & Services → Credentials
  2. Click + Create Credentials → OAuth client ID
  3. For Application type, choose Web application
  4. Give it a name (e.g. DAVE)
  5. Under Authorized JavaScript origins, click + Add URI and add:
    • https://drorlazar-sett.github.io — for the live site
    • http://localhost:7777 — for local development
  6. Click Create
  7. A popup shows your Client ID — copy it (it looks like 123456789-xxxxx.apps.googleusercontent.com)

Important: Make sure you pick Web application, not "Desktop app." Desktop app will not work with DAVE.

5 Enter the Client ID in DAVE
  1. Open DAVE and click the Settings button in the top bar
  2. In the Google Drive section, paste your Client ID
  3. Click Save
6 Sign in and browse!
  1. Click Source in the toolbar and choose Google Drive
  2. A Google sign-in popup will appear — pick your account and click Allow
  3. The folder browser opens with your Drive files. Navigate to a folder and click Load Files!

Your Google session lasts about 1 hour. When it expires, just click Source → Google Drive again to sign back in.

Once You're Connected

Browsing folders
  • Click any folder to go inside it
  • Use the breadcrumb trail at the top to go back
  • The footer shows how many viewable files are in the current folder
  • Check Subfolders to also grab files from nested folders
  • Use the depth dropdown to limit how deep (1, 2, 3 levels, or All)
  • Click Load Files to display everything in the DAVE grid
Quick loading with URLs

You can also paste or drag cloud URLs directly:

  • Search bar: Paste any S3 console URL, s3:// link, or Google Drive folder URL
  • Drag & drop: Drag a URL from your browser's address bar into DAVE

DAVE will respect your subfolder toggle setting — if it's set to "2", it loads files from 2 levels deep.

Supported file types

Cloud sources support everything local folders do:

3D Models — FBX, GLB Images — PNG, JPG, GIF, SVG, WebP... Video — MP4, WebM, MOV... Audio — MP3, WAV, OGG, FLAC... Fonts — TTF, OTF, WOFF, WOFF2 Text — TXT, JSON, XML, MD...

Your Data is Safe

DAVE runs 100% in your browser. Your credentials never leave your machine.

No server involved — DAVE talks directly to AWS / Google from your browser
Local storage only — credentials are saved in your browser, never uploaded
Read-only access — DAVE can only view files, never modify or delete them
Google tokens expire — Drive sessions auto-expire after 1 hour

Troubleshooting

AWS S3 Issues

"Cannot reach S3 bucket" or CORS error

This is the most common issue. It means your bucket's CORS settings aren't allowing DAVE to connect.

  • Go back to Step 1 and double-check the CORS configuration
  • Make sure you pasted the JSON exactly as shown (no missing brackets)
  • Try setting AllowedOrigins to ["*"] to rule out origin issues
  • After saving CORS changes, wait a minute and try again (AWS caches the old settings briefly)
"Invalid AWS Access Key ID"

Your Access Key ID doesn't match any key in AWS.

  • Open Settings and check for extra spaces before or after the key
  • Make sure you copied the Access Key ID (starts with AKIA), not something else
  • If you're unsure, create a new access key in the IAM console
"Access Denied"

Your IAM user doesn't have permission to read this bucket.

  • The user needs at least s3:ListBucket and s3:GetObject permissions
  • The easiest fix: attach the AmazonS3ReadOnlyAccess policy to your IAM user
  • Also verify the bucket name and region are correct in Settings

Google Drive Issues

Sign-in popup doesn't appear
  • Check that popups are not blocked for this site (look for a popup-blocked icon in your address bar)
  • Some ad blockers block the Google Sign-In library. Try disabling your ad blocker for DAVE
  • If you see "Google Sign-In library not loaded", the GIS script is being blocked
"redirect_uri_mismatch" error

This means the URL you're using DAVE from isn't listed in your OAuth client's authorized origins.

  • Go to Google Cloud Console → Credentials
  • Click on your OAuth Client ID
  • Under Authorized JavaScript origins, make sure you have:
    • https://drorlazar-sett.github.io (no trailing slash, no path)
    • http://localhost:7777 (for local use)
  • Also double-check you chose Web application (not "Desktop app") as the type
"Not configured" error
  • Open Settings (gear icon) and verify your Client ID is saved
  • It should end with .apps.googleusercontent.com
  • Make sure you clicked Save after pasting it
"Session expired"

Google Drive tokens last about 1 hour. This is normal.

  • Just click Source → Google Drive again to sign back in
  • Your settings and Client ID stay saved — you only need to re-authorize

General

Files not showing up
  • DAVE only displays files with recognized extensions (images, videos, 3D models, audio, fonts, text)
  • Check the filter settings (eye icon in the toolbar) — make sure the file type you need is enabled
  • If using subfolders, make sure the subfolder toggle isn't set to "off"
